## 关键改进点

### ✅ 1. 初始化时查询远程状态

**问题**：客户端重启后，不知道远程 Agent 是否正在等待权限

**解决**：在 `initialize()` 中调用 `_syncRemoteStateAndPermission()`

**效果**：
- 客户端重启后能恢复权限请求状态
- 多设备同步时能感知到其他设备的权限请求

### ✅ 2. 事件驱动的权限请求缓存

**问题**：远程模式下权限请求信息丢失

**解决**：在 `_handleAgentEvent()` 中处理 `toolPermissionRequest` 事件

**效果**：
- 实时接收远程权限请求
- 缓存权限请求，支持同步访问

### ✅ 3. 状态变更时主动查询

**问题**：状态变为 `waitingPermission` 时，可能错过事件

**解决**：在 `_handleStateChange()` 中添加 `waitingPermission` 状态处理

**效果**：
- 双重保障，确保权限请求不丢失
- 支持通过事件和状态两种方式获取权限请求

### ✅ 4. 同步方法支持远程模式

**问题**：`getPendingPermissionRequest()` 在远程模式返回 `null`

**解决**：从缓存中返回权限请求

**效果**：
- 保持同步 API，客户端无需修改
- 统一本地和远程模式的调用方式

## 使用场景

### 场景1：客户端重启，恢复权限请求

```
1. 远程 Agent 正在等待权限（用户在设备A发起了权限请求）
2. 用户在设备B重启客户端
3. initialize() 查询远程状态
4. 检测到 waitingPermission 状态
5. 查询并缓存权限请求
6. 客户端显示权限请求卡片 ✅
```

### 场景2：实时权限请求

```
1. 用户在客户端发送消息
2. 远程 Agent 需要权限
3. 广播 toolPermissionRequest 事件
4. 客户端接收事件并缓存
5. 立即显示权限请求卡片 ✅
```

### 场景3：网络中断后重连

```
1. 客户端网络中断
2. 远程 Agent 进入等待权限状态
3. 客户端网络恢复
4. 状态同步触发 onStateChanged
5. _handleStateChange 检测到 waitingPermission
6. 查询并缓存权限请求
7. 显示权限请求卡片 ✅
```

### 场景4：用户响应权限

```
1. 用户点击"允许"
2. respondToPermission() 发送决策到远程
3. 清除本地缓存的权限请求
4. 远程 Agent 继续执行
5. 状态变为 processing 或 idle
6. 客户端同步最新消息
```
